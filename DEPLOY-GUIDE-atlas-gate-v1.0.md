## 閘門運作邏輯
1. 頁面載入瞬間先隱藏全部內容，顯示「會員資格確認中…」
2. 依序檢查：
   - 本工作階段已驗證過 → 直接放行
   - 主站後台若開啟「全站限時開放」→ 視同黃金放行（沿用 gf_open_until RPC）
   - 本瀏覽器記過的通行碼 → 重新呼叫 gf_check_pass 驗證；**後台停用該碼，下次造訪即重新上鎖**
3. 皆未通過 → 顯示墨海藍品牌鎖定畫面：通行碼輸入框＋三個導流連結（會員方案／預約諮詢／回主站）
4. 輸入通行碼：L4 → 解鎖並記住；L1–L3 → 提示等級不足需黃金；無效 → 提示已停用
5. Supabase 連線失敗 → **預設上鎖**（fail-closed），不會因斷線而漏放

## 部署方式（GitHub 網頁 Upload files）
1. 進入 `FalconKenny/TW-US-power-atlas` → **Add file → Upload files**
2. 將解壓後的 4 個 html 與 `assets` 資料夾整個拖入（保持結構），Commit 覆蓋
3. 等 GitHub Pages 重建約 1–2 分鐘

## 驗收清單
- [ ] 無痕視窗直接開 https://falconkenny.github.io/TW-US-power-atlas/ → 只看到鎖定畫面，看不到任何地圖內容
- [ ] 直接開 /taiwan.html、/americas.html、/about.html → 同樣被鎖
- [ ] 輸入黃金通行碼 → 解鎖；換頁不需重新輸入；關瀏覽器重開仍記得（會靜默重新驗證）
- [ ] 輸入白銀通行碼 → 顯示「等級不足，需黃金會員」
- [ ] 在主站後台停用某黃金通行碼 → 該使用者下次開新工作階段即被重新上鎖
- [ ] 後台開啟「全站限時開放」→ 訪客免通行碼直接進入

## 誠實的安全邊界
這已擋掉「直接輸入網址」的一般訪客，且通行碼可由後台即時撤銷。但這仍是**前端閘門**：懂技術的人開瀏覽器開發者工具，仍可讀到頁面原始碼與 data-*.js 內的資料。若未來資料價值升高，下一階段是把各州/縣市資料搬進 Supabase 表格＋RLS，由通行碼換取資料（如同主站 gf_premium_list 的做法），屆時資料才真正不落地。目前階段此方案是免費、零後端改動下的最佳平衡。
